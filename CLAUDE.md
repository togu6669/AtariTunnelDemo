## Build

Assemble and produce the executable:

```
..\MADS\bin\windows_x86_64\mads.exe tunnel.asm -o:build/tunnel.xex
```

In VS Code: **Ctrl+Shift+B** runs this as the default build task.

Run in emulator (separate VS Code task):

```
..\Altirra\Altirra64.exe build/tunnel.xex
```

Tools (`mads.exe`, `Altirra64.exe`) live one directory above the repo root and are not checked in.

## Architecture

This is an Atari 8-bit demo (6502 assembly, MADS assembler) that displays a full-screen animated tunnel effect with music.

### Entry point: `tunnel.asm`

- Loads at `$2400` (offset by ~1 KB to avoid overlapping the RMT music player, which requires ~1 KB of memory below its `PLAYER` address).
- `START`: initialises music module, sets up the Antic display list and screen, installs the VBI handler, then loops calling `DRAW` every frame.
- `DRAW`: iterates the 40Ă—24 `DIST_TABLE`, adds the ever-decrementing `PHASE` byte, masks to 4 bits, looks up `SHADE_TABLE`, and writes the result to 8 interleaved destination row pointers that cover the bitmap. This creates the illusion of a scrolling tunnel.
- `DISPLAY_LIST`: Antic mode `$0E` (GRAPHICS 7 / 160Ă—192, 4-colour bitmap), split across two LMS entries (`$4010` and `$5000`) to span more than 4 KB of screen RAM.
- `SHADE_TABLE` / `SHADE_TABLE_HARD` / `SHADE_TABLE_SOFT` / `SHADE_TABLE_ALIAS`: three 16-entry presets mapping distance values (0â€“15) to 2bpp pixel pairs (`$00`, `$55`, `$AA`, `$FF`). Change `SHADE_TABLE` label to switch visual style.
- `DIST_TABLE`: 40Ă—24 bytes of pre-computed radial distance values, mod 16. Regenerate with `gendisttable.py`.
- `STEREOMODE = 0` in the `.proc music` block selects mono 4-track RMT playback.
- VBI (`MUSIC_VBI`): deferred VBI that calls `music.play` each frame, then jumps to `XITVBV`.

### Music subsystem (RMT â€” Raster Music Tracker)

Three files are included verbatim into `tunnel.asm`:

| File | Role |
|---|---|
| `music.feat` | Feature flags generated by RMT for the specific `.rmt` module; controls conditional assembly of the player. Re-export from RMT whenever the music module changes. |
| `rmt_player.asm` | RMT player v1.20090108 by Radek Sterba. Must be assembled at a page-aligned address (`PLAYER` = `$..00`). Occupies ~1 KB of memory below `PLAYER` for frequency tables and per-track variables. Do not edit. |
| `rmt_relocator.asm` | Helper macro `rmt_relocator` that embeds a `.rmt` binary and patches its internal pointers at assemble time. |

The music module is embedded via:
```
rmt_relocator 'music.rmt' modul
```

To change the music: swap `music.rmt`, re-export `music.feat` from RMT, then rebuild.
