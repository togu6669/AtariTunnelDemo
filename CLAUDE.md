# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build

Assemble and produce the executable:

```
..\MADS\bin\windows_x86_64\mads.exe tunnel.asm -o:build/tunnel.xex
```

In VS Code: **Ctrl+Shift+B** runs this as the default build task.

Run in emulator (separate VS Code task):

```
..\Altirra\Altirra64.exe build/tunnel.xex
```

Tools (`mads.exe`, `Altirra64.exe`) live one directory above the repo root and are not checked in.

## Architecture

This is an Atari 8-bit demo (6502 assembly, MADS assembler) that displays a full-screen animated tunnel effect with music.

### Entry point: `tunnel.asm`

- Loads at `$2400` (offset by ~1 KB to avoid overlapping the RMT music player, which requires ~1 KB of memory below its `PLAYER` address).
- `START`: initialises music module, sets up the Antic display list and screen, installs the VBI handler, then loops calling `DRAW` every frame.
- `DRAW`: iterates the 40×24 `DIST_TABLE`, adds the ever-decrementing `PHASE` byte, masks to 4 bits, looks up `SHADE_TABLE`, and writes the result to 8 interleaved destination row pointers that cover the bitmap. This creates the illusion of a scrolling tunnel.
- `DISPLAY_LIST`: Antic mode `$0E` (GRAPHICS 7 / 160×192, 4-colour bitmap), split across two LMS entries (`$4010` and `$5000`) to span more than 4 KB of screen RAM.
- `SHADE_TABLE` / `SHADE_TABLE_HARD` / `SHADE_TABLE_SOFT` / `SHADE_TABLE_ALIAS`: three 16-entry presets mapping distance values (0–15) to 2bpp pixel pairs (`$00`, `$55`, `$AA`, `$FF`). Change `SHADE_TABLE` label to switch visual style.
- `DIST_TABLE`: 40×24 bytes of pre-computed radial distance values, mod 16. Regenerate with `gendisttable.py`.
- `STEREOMODE = 0` in the `.proc music` block selects mono 4-track RMT playback.
- VBI (`MUSIC_VBI`): deferred VBI that calls `music.play` each frame, then jumps to `XITVBV`.

### Music subsystem (RMT — Raster Music Tracker)

Three files are included verbatim into `tunnel.asm`:

| File | Role |
|---|---|
| `music.feat` | Feature flags generated by RMT for the specific `.rmt` module; controls conditional assembly of the player. Re-export from RMT whenever the music module changes. |
| `rmt_player.asm` | RMT player v1.20090108 by Radek Sterba. Must be assembled at a page-aligned address (`PLAYER` = `$..00`). Occupies ~1 KB of memory below `PLAYER` for frequency tables and per-track variables. Do not edit. |
| `rmt_relocator.asm` | Helper macro `rmt_relocator` that embeds a `.rmt` binary and patches its internal pointers at assemble time. |

The music module is embedded via:
```
rmt_relocator 'music.rmt' modul
```

To change the music: swap `music.rmt`, re-export `music.feat` from RMT, then rebuild.

### Memory map

| Address | Contents |
|---|---|
| `$2400` | Program start (`.proc music` / player area begins here) |
| `$2400` + RMT offset | RMT player (`PLAYER` must be page-aligned) |
| `$4010`–`$4FFF` | Bitmap screen RAM (first half) |
| `$5000`–`$5FFF` | Bitmap screen RAM (second half) |
| `$02E0` | Atari run address vector → `START` |

### Helper script

`gendisttable.py` — Python script that prints a 40×24 `.BYTE` table of `floor(sqrt(dx²+dy²)) % 16` values. Copy its output into `DIST_TABLE` in `tunnel.asm` when tweaking tunnel geometry.

### `test/` directory

Contains earlier experimental versions of the effect (character-based, single-colour, escape-key exit). Not assembled by the main build.

## `man_scroll.asm` — sprite coarse-scroll experiment

Standalone program (no music, no tunnel) that draws a man sprite on a GR.7 (mode `$0E`) bitmap and coarse-scrolls it.

- Loads at `$2000`. Run address vector at `$02E0` → `ANIM_MAIN`.
- Build: `mads.exe man_scroll.asm -o:build/man_scroll.xex`

### Display list (`ANIM_DLIST`, assembled at `$2300`)

Antic mode `$0E` (160×192, 4-colour), split across two LMS entries:

| LMS label | Default address | Rows covered |
|---|---|---|
| `SCRN_LO_DL` | `SCRN_LO` = `$4010` | 0–101 (102 rows) |
| `SCRN_HI_DL` | `SCRN_HI` = `$5000` | 102–191 (90 rows) |

The split is necessary because 192 mode-E rows × 40 bytes = 7680 bytes, which crosses a 4 KB ANTIC boundary. `SCRN_SPLIT = 102` is the first row stored in the high half.

DMA is disabled (`SDMCTL = $00`) during setup, then re-enabled with `LDA #$22 : STA SDMCTL` before the main loop.

### Coarse scrolling (`coarse_scroll_down`)

Scrolls the viewport one pixel row down by adding **40** (one mode-E row width) to each LMS address in the display list. Both `SCRN_LO_DL` and `SCRN_HI_DL` are updated as full 16-bit adds (with carry propagation to the high byte). Adding only 1 instead of 40 is a common mistake — it shifts by one byte, not one row.

### Row address table

`INIT_ROW_TABLE` pre-computes `ROW_TBL_LO` / `ROW_TBL_HI` (192 entries, stride 40) so `DRAW_SPR_FULL` and `CLEAR_SCREEN` can address any screen row without on-the-fly multiplication.

### Memory map (`man_scroll.asm`)

| Address | Contents |
|---|---|
| `$00B0`–`$00BF` | Zero-page variables (MAN_X, MAN_YL, accumulators, scratch) |
| `$2000` | `ANIM_MAIN` entry point |
| `$2300` | `ANIM_DLIST` display list |
| `$2400` | `ROW_TBL_LO` (192 bytes) |
| `$24C0` | `ROW_TBL_HI` (192 bytes) |
| `$2580` | Sprite data (`graphics/sprite_data.asm`) |
| `$4010`–`$4FFF` | Screen RAM low half |
| `$5000`–`$5E0F` | Screen RAM high half |
